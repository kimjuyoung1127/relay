# docs/ref/ 거버넌스 규칙

## 용도
안정적, 장기 참조 문서 보관소.

## 규칙
- 새 참조 문서는 `ref/`에만 생성
- 운영 상태 내용은 여기에 작성하지 않음 → `status/` 사용
- 아키텍처, 코드 패턴, 운영 모델 등 느리게 변하는 문서만 보관
- 문서 간 링크 우선, 내용 중복 지양

## 현재 문서
- `ARCHITECTURE.md` — 시스템 구조, 상태 모델, 프로바이더 모델
- `CODE-REVIEW-GRAPH-TUNING.md` — 그래프 기반 스코프 분석 설정
- `DOCS-OPERATING-MODEL.md` — 문서 유지보수 규칙
